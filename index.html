应用解决什么问题

帮助已经拥有第三方 AI API 账号的用户，把多个模型供应商统一收纳到一个隐私优先的客户端中，集中完成 API 配置、聊天、模型切换、历史管理和本地/iCloud 同步。

目标用户

开发者、AI 深度用户、需要跨平台管理多个模型 API 的个人用户，以及不希望把聊天内容托管到开发者服务器的隐私敏感用户。

审核时必须明确的一点

审核员如果没有可用的 AI 测试凭据，就无法完成核心功能验证。你需要为审核准备一套真实可用、限额可控的测试 API 凭据。

敏感权限与设备能力

通知

用于提醒 AI 回复完成或关键状态变化。现在在首次引导页点击“允许”会触发 iOS 系统通知授权弹窗。

照片库

用于选择图片并发送给支持图片理解的模型。首次引导页点击“允许”会触发系统照片权限弹窗。

Face ID / Touch ID

用于访问敏感数据前的本地认证。首次引导页点击“允许”会触发系统生物识别验证。

iCloud

用于 CloudKit 同步聊天、资料和配置。它不是传统的系统授权弹窗型权限，但需要设备已登录 iCloud。

当前项目中已配置的用途说明包括：NSFaceIDUsageDescription 与 NSPhotoLibraryUsageDescription。通知权限不依赖 Info.plist purpose string，而是通过系统授权 API 请求。

外部服务 / 平台

Apple AuthenticationServices

用于 Sign in with Apple。

Apple StoreKit

用于非消耗型永久解锁购买与恢复购买。

Apple CloudKit / iCloud

用于用户数据同步，不是开发者自建数据库。

第三方 AI 平台

例如 OpenAI、Anthropic、Google Gemini、xAI、DeepSeek、阿里云、智谱、Kimi、MiniMax 等，由用户自行持有 API 账号。

审核风险提示： 如果 Review Notes 不提供可用测试 API 凭据，审核员通常无法验证核心聊天能力，极容易再次触发 Guideline 2.1。

区域与内容差异说明

功能一致性

应用 UI、买断模式、登录方式、权限流程、设置页、政策页在各地区保持一致。

实际可用性差异

可连接的第三方 AI 平台受用户自己的 API 账号、第三方平台地区政策、网络条件和平台服务可用性影响。你在审核备注里应这样写，而不是简单写“所有地区完全一致”。

App purpose:
AI Nexus is a privacy-first client app for users who already have accounts with third-party AI providers. The app helps users securely store their own API configurations, switch between multiple model providers, keep local conversation history, and optionally sync app data through iCloud. AI Nexus does not operate its own AI backend and does not host user conversations on developer-owned servers.

Main review flow:
1. Launch the app.
2. Complete onboarding and review the permission prompts.
3. Sign in with Apple.
4. Add the test AI configuration below.
5. Start a chat and send a test prompt.
6. Open the Me tab to review account details, permissions, policies, feedback/reporting, and purchase information.
7. Open the purchase screen to review the lifetime unlock and restore purchase flow.

你提交前还必须亲自完成的事项

1. 准备审核专用 AI 凭据

必须提供真实可用的 API Key、Endpoint、Model，并限制额度，避免审核无法完成核心功能测试。

2. 录制真机视频

从桌面启动开始，全流程展示权限、登录、聊天、举报、购买或恢复购买。

3. 补全 App Store Connect 备注

把上面的审核备注、外部服务清单、区域说明、测试账号信息一次性写完整。

4. 准备公网政策链接

隐私政策和支持页面最好都提供公网 URL，不要只依赖应用内页面。

5. 检查线上 CloudKit 与 IAP

CloudKit 需要部署生产环境 Schema；IAP 需要在 App Store Connect 创建并处于可审核状态。

6. 再做一次真机回归

重点验证首次引导、权限弹窗、Apple 登录、iCloud 同步、买断购买、恢复购买。
